= flume-kafka

== 1.flume配置

=== 1.1.新建flume配置文件

flume/carjob/目录下新建flume-kafka.conf文件

内容如下：

----
a1.sources = r1
a1.channels = c1
a1.sinks = k1 k2

a1.sources.r1.type = spooldir
a1.sources.r1.spoolDir = /root/cardata
a1.sources.r1.fileHeader = true
a1.sources.r1.fileSuffix = .COMPLETED

a1.channels.c1.type = memory
a1.channels.c1.capacity = 1000
a1.channels.c1.transactionCapacity = 100

a1.sinks.k1.type = org.apache.flume.sink.kafka.KafkaSink
a1.sinks.k1.kafka.bootstrap.servers = 10.7.121.58:9092
a1.sinks.k1.topic = cardata-topic
a1.sinks.k1.batchSize = 1
a1.sinks.k2.type = logger

a1.sources.r1.channels = c1
a1.sinks.k1.channel = c1
a1.sinks.k2.channel = c1
----

本配置文件监听/root/cardata目录下的文件，并将数据传输给kafka


== 2.kafka配置

=== 2.1.新建kafka配置文件

kafka/config目录下新建server.properties-java文件

内容如下：

----
#表示Kafka Broker的唯一标识，每个Broker的ID应该唯一
broker.id=1
#声明Kafka监听的地址和端口，用于与客户端或其他Broker通信
listeners=PLAINTEXT://10.7.121.58:9092
#网络处理线程数，用于接收和处理客户端的请求
num.network.threads=3
#磁盘I/O线程数，用于执行磁盘读写操作
num.io.threads=8
#定义套接字发送缓冲区的大小，该值默认为100KB
socket.send.buffer.bytes=102400
#定义套接字接收缓冲区的大小，该值默认为100KB
socket.receive.buffer.bytes=102400
#定义接收请求的最大字节数，该值默认为100MB
socket.request.max.bytes=104857600
#声明Kafka Broker的日志目录
log.dirs=/kafkalogs/kafka-logs
#设置Kafka topic的初始分区数
num.partitions=1
#定义每个数据目录恢复线程数的数量
num.recovery.threads.per.data.dir=1
#设置存储消费者offset的topic的副本数量
offsets.topic.replication.factor=1
#设置事务状态日志的副本数量
transaction.state.log.replication.factor=1
#设置事务状态日志的最小ISR值，即必须有多少个副本在活跃状态下才能继续执行事务
transaction.state.log.min.isr=1
#配置Log消息的保留时间，单位为小时
log.retention.hours=168
#设置单个日志文件的最大大小，单位字节
log.segment.bytes=1073741824
#配置检查保留策略的时间间隔，单位为毫秒
log.retention.check.interval.ms=300000
#设置Zookeeper注册中心的连接地址
zookeeper.connect=localhost:2181
#设置Zookeeper连接的超时时间，单位为毫秒
zookeeper.connection.timeout.ms=6000
#定义消费者组的初始化均衡的延迟时间，单位为毫秒
group.initial.rebalance.delay.ms=0
----

使用该配置文件可以启动一个Kafka Broker并使其能够与客户端应用进行通信

=== 2.2.新建zookeeper配置文件

zookeeper/conf目录下修改zoo.cfg文件

内容如下：

----
# The number of milliseconds of each tick
tickTime=2000
# The number of ticks that the initial
# synchronization phase can take
initLimit=10
# The number of ticks that can pass between
# sending a request and getting an acknowledgement
syncLimit=5
# the directory where the snapshot is stored.
# do not use /tmp for storage, /tmp here is just
# example sakes.
dataDir=/tmp/zookeeper
# the port at which the clients will connect
clientPort=2181
----

== 3.启动flume-kafka组件

=== 3.1.启动zookeeper

启动命令：

[source,shell]
----
bin/zkServer.sh start
----

=== 3.2.启动kafka,创建主题

启动命令：

[source,shell]
----
bin/kafka-server-start.sh config/server.properties-java
----

创建名为cardata-topic的主题：

[source,shell]
----
bin/kafka-topics.sh --create --topic cardata-topic --partitions 1 --replication-factor 1 --zookeeper localhost:2181
----

查看现有主题列表：

[source,shell]
----
bin/kafka-topics.sh --list --zookeeper 10.7.121.58:2181
----


=== 3.3.启动flume

启动命令：

[source,shell]
----
bin/flume-ng agent --conf conf/ --conf-file carjob/flume-kafka.conf --name a1 -Dflume.root.logger=INFO,console
----

== 4.验证kafka是否成功消费

cardata目录下新建test.txt文件

查看kafka消费情况

[source,shell]
----
bin/kafka-console-consumer.sh --bootstrap-server 10.7.121.58:9092 --topic cardata-topic --from-beginning
----

_--from-beginning_：从topic最早的消息进行消费
